# مستندات موجودیت‌های مرتبط با بخش فروشگاه

این سند شامل تحلیل جامع تمامی مدل‌ها و موجودیت‌های موجود در پروژه است که می‌توانند به صورت مستقیم یا غیرمستقیم در پیاده‌سازی بخش فروشگاه مورد استفاده قرار گیرند. این تحلیل برای تسهیل فرآیند اضافه شدن بخش فروشگاهی تهیه شده است.

---

## ۱. موجودیت‌های موجود و نقش آن‌ها (Existing Entities)

### الف) مدیریت کاربران و احراز هویت (Users & Verification)
زیرساخت اصلی شناسایی و اعتبارسنجی مشتریان.

*   **مدل User (در اپلیکیشن `users`):**
    *   **فیلدهای کلیدی:** `phone_number`, `score`, `rank`, `is_phone_verified`.
    *   **کاربرد:** خریدار اصلی. امتیاز کاربر (`score`) می‌تواند برای اعمال تخفیف‌های وفاداری یا سطوح دسترسی استفاده شود.
*   **مدل Referral (در اپلیکیشن `users`):**
    *   **فیلدهای کلیدی:** `referrer`, `referred`.
    *   **کاربرد:** پیاده‌سازی سیستم‌های "دعوت از دوستان" برای اهدای بن خرید یا درصد سود به معرف در صورت خریدِ کاربر معرفی شده.
*   **مدل InGameID (در اپلیکیشن `users`):**
    *   **فیلدهای کلیدی:** `game`, `player_id`.
    *   **کاربرد:** در فروشگاه‌های گیمینگ، برای تحویل محصولات دیجیتال مستقیم به اکانت بازی کاربر (مانند واریز جم یا یوسی) حیاتی است.
*   **مدل Verification (در اپلیکیشن `verification`):**
    *   **فیلدهای کلیدی:** `level`, `is_verified`.
    *   **کاربرد:** محدود کردن سقف خرید یا دسترسی به محصولات خاص برای کاربرانی که احراز هویت سطح بالا انجام داده‌اند.

### ب) زیرساخت مالی و تراکنش‌ها (Wallet)
قلب تپنده بخش فروشگاه برای مدیریت وجوه.

*   **مدل Wallet:**
    *   **فیلدهای کلیدی:** `total_balance`, `token_balance`.
    *   **کاربرد:** کیف پول چند منظوره (ریالی و توکنی) برای پرداخت هزینه سفارش‌ها.
*   **مدل Transaction:**
    *   **فیلدهای کلیدی:** `amount`, `transaction_type`, `status`, `order_id`, `authority`.
    *   **کاربرد:** ثبت دقیق تاریخچه مالی خریدها. فیلد `order_id` می‌تواند مستقیماً به شناسه سفارش فروشگاه متصل شود.
*   **مدل Refund:**
    *   **فیلدهای کلیدی:** `amount`, `status`, `refund_id`.
    *   **کاربرد:** مدیریت خودکار بازگرداندن وجه در صورت لغو سفارش یا مرجوعی کالا.

### ج) مدیریت محتوا و رسانه (Blog & Common)
استفاده از زیرساخت‌های موجود برای نمایش اطلاعات محصولات.

*   **مدل Media (در اپلیکیشن `blog`):**
    *   **فیلدهای کلیدی:** `url`, `type` (image/video/file), `alt_text`.
    *   **کاربرد:** مدیریت تصاویر گالری محصول، ویدیوهای بررسی و کاتالوگ‌های PDF.
*   **مدل Category & Tag (در اپلیکیشن `blog`):**
    *   **فیلدهای کلیدی:** `slug`, `name`, `parent`.
    *   **کاربرد:** دسته‌بندی درختی محصولات و برچسب‌گذاری برای فیلتر کردن و بهبود سئو.
*   **مدل Comment & Reaction (در اپلیکیشن `blog`):**
    *   **کاربرد:** پیاده‌سازی بخش نقد و بررسی محصولات و امتیازدهی کاربران به کالاها.

### د) خدمات و مسابقات (Tournaments)
برخی خدمات فعلی خود نوعی محصول هستند.

*   **مدل Tournament:**
    *   **فیلدهای کلیدی:** `entry_fee`, `is_token_based`, `prize_pool`.
    *   **کاربرد:** شرکت در تورنمنت‌ها خود یک "محصول خدماتی" است که از درگاه پرداخت و کیف پول استفاده می‌کند.
*   **مدل Rank:**
    *   **فیلدهای کلیدی:** `required_score`.
    *   **کاربرد:** می‌توان محصولات را بر اساس رتبه کاربر قیمت‌گذاری متفاوت کرد (مثلاً تخفیف ویژه برای رتبه‌های طلایی).

### ه) ارتباطات و پشتیبانی (Support & Chat)
*   **مدل Ticket:** مدیریت درخواست‌های گارانتی و سوالات قبل از خرید.
*   **مدل Notification:** اطلاع‌رسانی خودکار تغییر وضعیت سفارش (از "ثبت شده" تا "ارسال شده").
*   **مدل Message & Conversation:** چت زنده با پشتیبان فروشگاه برای مشاوره خرید.

## ۲. موجودیت‌های جدید پیشنهادی (New Shop Models)

برای تکمیل این بخش، ایجاد یک اپلیکیشن جدید به نام `shop` با مدل‌های زیر پیشنهاد می‌شود:

| مدل | توضیحات و فیلدها |
| :--- | :--- |
| **Product** | شامل `name`, `slug`, `base_price`, `sale_price`, `stock_count`, `product_type` (فیزیکی/دیجیتال). |
| **Order** | شامل `user`, `total_price`, `discount_amount`, `status` (در انتظار، پرداخت شده، لغو شده، ارسال شده)، `tracking_code`. |
| **OrderItem** | جزئیات هر ردیف سفارش شامل `product`, `quantity`, `price_at_purchase`. |
| **DiscountCoupon** | شامل `code`, `percent`, `max_limit`, `expiry_date`, `min_order_value`. |
| **UserAddress** | شامل `province`, `city`, `address_detail`, `postal_code`, `receiver_name`, `receiver_phone`. |

## ۳. تحلیل مسیر پیاده‌سازی (Implementation Path)

1.  **اتصال به کیف پول:** استفاده از متد‌های موجود در اپلیکیشن `wallet` برای کسر موجودی پس از تایید نهایی سبد خرید.
2.  **یکپارچگی با رسانه:** استفاده از مدل `Media` برای تمام تصاویر محصولات جهت جلوگیری از تکرار کد و بهینه‌سازی حجم تصاویر.
3.  **سیستم اعلان:** فراخوانی سرویس‌های `notifications` در هر تغییر وضعیت سفارش (مثلاً ارسال کد رهگیری پستی).
4.  **تحویل خودکار:** برای محصولات دیجیتال، استفاده از `InGameID` برای ارسال مستقیم آیتم به بازی (در صورت وجود API بازی).

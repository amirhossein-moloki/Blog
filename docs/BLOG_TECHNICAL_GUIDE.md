# مستندات فنی ماژول وبلاگ (Blog Module)

این سند به بررسی جزئیات پیاده‌سازی، تکنولوژی‌های استفاده شده و نحوه عملکرد بخش وبلاگ پروژه می‌پردازد.

## ۱. تکنولوژی‌های مورد استفاده (Technology Stack)

*   **Django & Django Rest Framework (DRF):** فریم‌ورک اصلی برای مدیریت منطق بک‌اند و ارائه APIها.
*   **PostgreSQL:** پایگاه داده رابطه‌ای برای ذخیره‌سازی داده‌ها.
*   **Celery & Redis:** برای مدیریت تسک‌های ناهمزمان (Asynchronous Tasks) مانند:
    *   انتشار پست‌های زمان‌بندی شده.
    *   ارسال نوتیفیکیشن‌های ایمیلی برای نظرات جدید.
*   **Django CKEditor 5:** ویرایشگر متن پیشرفته برای مدیریت محتوای پست‌ها و نظرات.
*   **Jalali Date:** برای نمایش و مدیریت تاریخ‌ها به شمسی (استفاده از کتابخانه `django-jalali-date`).
*   **Pillow:** برای پردازش تصاویر و تبدیل آن‌ها به فرمت بهینه **AVIF**.

---

## ۲. معماری و ساختار داده‌ها (Data Models)

مدل‌های اصلی این ماژول عبارتند از:

*   **Post (پست):** هسته اصلی وبلاگ شامل عنوان، محتوا، اسلاگ، وضعیت (Draft, Published, Scheduled)، و تنظیمات سئو.
*   **AuthorProfile (پروفایل نویسنده):** توسعه مدل کاربر برای ذخیره اطلاعات نویسندگی.
*   **Media (رسانه):** مدیریت متمرکز فایل‌های آپلودی. تصاویر به‌طور خودکار به فرمت AVIF تبدیل می‌شوند تا سرعت بارگذاری افزایش یابد.
*   **Category & Tag:** برای دسته‌بندی و برچسب‌گذاری پست‌ها.
*   **Comment (نظرات):** سیستم نظرات تودرتو (Nested) با قابلیت تایید توسط ادمین.
*   **Reaction (واکنش‌ها):** سیستم لایک و اموجی که با استفاده از Generic Foreign Key به پست‌ها و کامنت‌ها متصل می‌شود.
*   **Series (مجموعه‌ها):** برای گروه‌بندی پست‌های مرتبط (مثل یک دوره آموزشی).

## ۳. ویژگی‌های کلیدی و نحوه کارکرد (Key Features & Workflow)

### الف) چرخه حیات پست (Post Life-cycle)
پست‌ها می‌توانند در وضعیت‌های مختلفی باشند:
1.  **Draft (پیش‌نویس):** فقط برای نویسنده و مدیران قابل مشاهده است.
2.  **Scheduled (زمان‌بندی شده):** پستی که برای انتشار در آینده تنظیم شده. یک تسک Celery (`publish_scheduled_posts`) به صورت دوره‌ای اجرا شده و این پست‌ها را در زمان مقرر منتشر می‌کند.
3.  **Published (منتشر شده):** برای همه کاربران قابل مشاهده است.

### ب) بهینه‌سازی کوئری‌ها (Performance Optimization)
برای جلوگیری از مشکل N+1، از `select_related` برای روابط One-to-One و ForeignKey و از `prefetch_related` برای روابط Many-to-Many و معکوس استفاده شده است. همچنین تعداد لایک‌ها و کامنت‌ها مستقیماً در کوئری اصلی با استفاده از `annotate` محاسبه می‌شوند.

### ج) مدیریت رسانه و سئو
*   هنگام ذخیره پست، سیستم به طور خودکار تصاویر موجود در متن (`content`) را شناسایی کرده و آن‌ها را به مدل `PostMedia` متصل می‌کند.
*   فیلدهای اختصاصی برای `SEO Title` و `SEO Description` و `OG Image` برای بهینه‌سازی در موتورهای جستجو و شبکه‌های اجتماعی تعبیه شده است.

### د) سیستم نظرات و نوتیفیکیشن
نظرات جدید پس از ثبت در وضعیت `pending` قرار می‌گیرند. پس از ثبت هر نظر، یک تسک Celery برای اطلاع‌رسانی به نویسنده پست اجرا می‌شود.

### ه) محاسبه زمان مطالعه
زمان مطالعه (`reading_time_sec`) به صورت خودکار در متد `save` مدل `Post` بر اساس تعداد کلمات محتوا محاسبه می‌شود.

## ۴. اندپوینت‌های API (API Endpoints)

*   `GET /api/blog/posts/`: لیست پست‌ها (با قابلیت فیلتر بر اساس دسته‌بندی، برچسب، وضعیت "داغ" و جستجو).
*   `GET /api/blog/posts/{slug}/`: جزئیات کامل یک پست (تعداد بازدید در هر بار مشاهده افزایش می‌یابد).
*   `POST /api/blog/posts/`: ایجاد پست جدید (مخصوص نویسندگان و ادمین).
*   `GET /api/blog/posts/{slug}/comments/`: دریافت نظرات تایید شده یک پست.
*   `POST /api/blog/posts/{slug}/publish/`: انتشار دستی یک پست.
*   `GET /api/blog/posts/{slug}/related/`: دریافت پست‌های مرتبط بر اساس برچسب‌های مشترک.

---

## ۵. نحوه استفاده برای توسعه‌دهندگان

1.  **افزودن پست:** ابتدا باید یک `AuthorProfile` برای کاربر ایجاد شده باشد.
2.  **آپلود تصویر:** از اندپوینت `/api/blog/media/` برای آپلود تصاویر استفاده کنید و سپس `id` رسانه را در `cover_media` یا در متن اصلی قرار دهید.
3.  **فیلتر فیلدها:** با استفاده از پارامتر `?fields=title,slug` در URL، می‌توانید فقط فیلدهای مورد نیاز خود را از API دریافت کنید تا حجم پاسخ کاهش یابد.
